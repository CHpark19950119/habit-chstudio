## 각 파일의 역할

### `박천홍_인생로드맵_v13.html` — 현재 아카이브 본체

- **상태**: 완성. 수정은 v14로 가면서 구조 개편 예정.
- **구성**: 13개 탭. 클릭하면 해당 섹션만 보이는 SPA 구조.
- **탭 목록**: 통합 타임라인 / 거주지 / 인물 사전 / 연애사 / 인터넷 여성들 / 전수 분석 / 심리 분석 / 공시 분석 / 경제활동 / 성적 심리 / 육성 기록 / 소설 콘티 / 미완료 과제.
- **CSS 변수**: `--bg:#f5f0e8`, `--ac:#8b2500`(아크센트 적갈색), `--ac2:#2d5a7b`, `--gn:#2d6b3f`, `--rd:#9b1b1b` 등. 세리프 폰트는 Noto Serif KR.
- **편집 시**: 전체 교체보다 특정 섹션만 `str_replace`로 수정하는 것이 안전. 먼저 `grep -n "섹션명"`으로 위치 파악 후 `view` + `view_range`로 구간 확인.

### `박천홍_인생로드맵_v12.html` — 보관

- **상태**: 이전 세션 결과물. v13 제작의 베이스.
- **편집 금지**: 참조용으로만 열기. 수정하지 않는다.

### `허락_전반_01-05.html` — 소설 제1부 전반 5챕터

- **챕터**: 01 754 / 02 이사 / 03 사마귀 / 04 찬양 / 05 바닥.
- **크기가 큰 이유**: 챕터마다 base64 인라인 이미지가 삽입됨. HTML 자체는 작지만 이미지 때문에 775KB.
- **편집 시**: `view_range`로 챕터별 구간을 잘라서 봐야 함. 이미지 블록(`data:image/jpeg;base64,/9j/...`)은 수십 KB짜리가 통째로 한 줄에 있으니 그 줄은 건너뛰고 작업.

### `허락_후반_06-11.html` — 소설 제1부 후반 6챕터

- **챕터**: 06 바깥 원 / 07 목격 / 08 MB / 09 딴지 / 10 잉꼬 / 11 수능.
- **편집 시**: 각 챕터 시작 `<div class="chapter" data-chapter="N">`에서 구분. 챕터 헤더 구조는 전반과 동일.

## 파일 편집의 표준 절차

1. **항상 view 먼저**. 파일 크기가 크면 `view_range`로 구간을 잡아서 읽는다.
2. **대용량 HTML**은 `grep -n "탭명\|섹션명\|키워드"`로 줄 번호를 먼저 잡고, 해당 구간만 `view_range`로 확인한다.
3. **수정은 `str_replace`** 로. 전체 재작성이 아니라 특정 블록만 교체.
4. **str_replace 실패 시**: 먼저 다시 `view`로 정확한 현재 텍스트를 확인. 공백·줄바꿈·base64 이미지 때문에 예상과 다를 수 있음.
5. **여러 블록 수정**: 하나씩 순차 실행. 한 번에 여러 `str_replace`를 묶지 말 것.

## 이미지 처리 규칙

소설에 이미지를 삽입할 때:
- PIL로 `convert('RGB')` 후 JPEG 저장.
- max 600px width.
- quality=60.
- base64 인라인 임베딩.

예시 코드:
```python
from PIL import Image
import base64, io

img = Image.open(path).convert('RGB')
img.thumbnail((600, 600))
buf = io.BytesIO()
img.save(buf, 'JPEG', quality=60)
b64 = base64.b64encode(buf.getvalue()).decode()
html_tag = f'<img src="data:image/jpeg;base64,{b64}" alt="...">'
```
