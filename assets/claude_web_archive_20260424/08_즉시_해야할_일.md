# 08. 즉시 해야 할 일

다음 세션(이 이식 자료를 들고 Claude Code가 작업을 시작할 때)의 우선순위. 천홍의 합의 없이 이 순서를 바꾸지 않는다.

## 세션 시작 절차

1. `CLAUDE.md` 읽기 (자동).
2. `01_인물.md` ~ `07_세션_히스토리.md` 순차 읽기.
3. 이 파일 `08_즉시_해야할_일.md` 읽기.
4. 천홍에게 호기 확인: "시즌 몇 호야?"
5. 천홍이 방향 제시하기 전까지 작업 시작하지 않는다.

## 최우선 3축 (선택)

천홍에게 다음 중 하나를 고르게 한다. Claude가 임의로 시작하지 않는다.

### 축 A. 소설 전용 핸드오프 문서 완성

**필요**: 기존 핸드오프는 전체 프로젝트를 다룬다. 소설 집필만을 위한 전용 핸드오프가 없음. 메모리에 "소설 전용 핸드오프 미완성, 다음 세션 최우선" 명시됨.

**작업 내용**:
- 소설 「허락」 22챕터 콘티를 문서화.
- 제1부 완성본의 문체 샘플 3~5개 추출.
- 제1부에서 수정이 필요한 챕터 목록 작성 (이예지 기록 수정 반영 필요, 박진리 원형 해제 반영 필요).
- 제2부·제3부 집필 착수 조건 명시.
- 인물/사건/거주지 참조 테이블.

**산출물**: `/mnt/user-data/outputs/핸드오프_소설_허락.md`.

**예상 시간**: 짧게 30분, 길게 90분.

### 축 B. 정다영 2024.05~07 불륜 구체적 경위 인터뷰

**필요**: 4호에서 구조는 나왔으나 경위가 미완. 6호·6.1호에서 반복 연기됨. 이 인터뷰 없이는 v14 제작 불가(심리 분석 탭의 결정적 누락).

**질문 순서 제안**:
1. 어떻게 알게 됐어? 다영이 직접 고백했어, 아니면 발견했어?
2. 알게 된 순간 어디 있었고 뭘 느꼈어?
3. 남자 2명이 누구였어? 각각 어떤 관계?
4. 시기(2024.05~07) 중 정확히 언제부터 언제까지?
5. 네가 알기 전에 너한테 어떤 신호가 있었어?
6. 고백 후 어떻게 관계를 재편했어?
7. 지금도 이 사건이 관계에 어떻게 작동하고 있어?

**주의**: 한 번에 2개 이상 질문하지 않는다. 천홍이 "모르겠다"고 하면 그대로 기록. 감정이 강하게 올라오면 쉼.

**산출물**: 인터뷰 기록 + v14 반영 + 소설 제3부 17~19장 재료.

### 축 C. v14 아카이브 착수

**필요**: v13은 단일 HTML 한계에 도달. 다중 페이지 + 데이터 시각화 + 인터랙티브 필요.

**착수 순서**:
1. 파일 구조 확정 (`04_프로젝트_파일.md`의 제안 참고 → 천홍 수정).
2. `index.html` (진입 + 통합 타임라인) 프로토타입.
3. 거래내역 JSON 추출 (`전체_거래내역_통합_1.csv` → 가짜지출 필터링 → `transactions.json`).
4. 통합 타임라인 컴포넌트: 1995–2026 슬라이더, 공적 위치 / 연애 / 거주 / 경제 4축 동시 추적.
5. 기존 13탭 중 거주지·인물 사전 먼저 페이지 분리.

**주의**: 전체를 한 세션에 완성하려 하지 않는다. 프로토타입 → 천홍 리뷰 → 조정 → 다음 페이지 순으로.

**산출물**: `/mnt/user-data/outputs/v14/` 폴더. 단계별 산출.

## 중기 과제

다음 세션에서 당장 다루지 않더라도 놓치지 않아야 하는 것.

### C1. 5급 3축(동기/방법/자격감) 분해

- 6호에서 천홍 "전부 다"로 답. 이후 해체 작업 중단됨.
- 세 축을 한 번에 파지 않는다. 한 세션에 한 축씩.
- 자격감 축이 가장 취약 — 마지막으로.

### C2. 거래내역 전수 인터뷰 2023~2026

- 2022까지 완료. 2023년은 참치어장 + JLPT + 졸업.
- 2024년은 공시 전업 시작 + 정다영 불륜 사건.
- 2025년은 7급 86.66 합격 / 5급 70 / 외영 90.66.
- 2026년은 현재 진행.
- 연도별 하나씩 세션 배분.

### C3. 유년기(0~11세) 기억 복원 인터뷰

- 6.1호에서 천홍이 지정.
- 감각·사물 기반 질문: 맛, 냄새, 촉감, 소리.
- 가족 인터뷰 가능성 (부모·누나).
- 군포/당동 거주지 재방문 — 실제 물리적 방문 권장(사진 기록).

### C4. 포르노 기록 시스템 실제 운용

- 설계는 완료. 실제 기록이 시작됐는지 Claude는 알 수 없음.
- 천홍에게 확인 필요: "기록 시작했어? 얼마나 누적됐어?"
- 누적되면 패턴 분석 가능 — A축/B축 비율, 시간대, 트리거 연결.

## 장기 과제

- **소설 제2부 「유리」 집필 착수** — 콘티 확정.
- **소설 제3부 「허락」 집필 착수** — 5급 결과에 따라 마지막 챕터의 방향 결정.
- **v14 완전 이행 후 v13 보존 처리**.
- **이식 자료 업데이트** — 세션이 누적되면 이 8개 문서도 갱신 필요.

## 이식 자료 자체의 갱신 주기

이 8개 문서는 일회성이 아니다. 프로젝트가 진행되면 **세션마다 갱신**되어야 한다.

- `01_인물.md`: 새 인물 확정, 기존 인물의 수정된 사실 반영.
- `02_현황과_목표.md`: 시험 결과, 공부 환경 변화.
- `03_심리_구조.md`: 새 패턴 확정(23번부터).
- `04_프로젝트_파일.md`: v14 진행 상태, 새 파일.
- `05_도메인_사전.md`: 새 상호명, 새 용어.
- `06_작업_원칙.md`: 원칙 자체가 바뀌면(드물게).
- `07_세션_히스토리.md`: 매 세션 말미 추가.
- `08_즉시_해야할_일.md`: 매 세션 시작·종료 시 갱신.

갱신은 **핸드오프 제작의 일부**로 본다.

## 현 세션 (이 이식 자료를 만든 세션)의 종결

이 세션은 인프라 세션이다. 끝날 때:

1. 8개 문서를 `/mnt/user-data/outputs/`에 두고 `present_files`.
2. 천홍이 다운로드 → Claude Code 작업 디렉토리(프로젝트 루트 혹은 `.claude/`)에 배치.
3. Claude Code에서 첫 세션 시작 시 `CLAUDE.md`가 자동으로 읽힘.

## 이식 자료가 놓치고 있을 수 있는 것

이 자료는 13,315건 거래내역 원본까지는 포함하지 않는다. Claude Code가 거래 분석을 할 때는 `/mnt/project/전체_거래내역_통합_1.csv` 또는 `전체_거래내역_통합.xlsx`를 직접 파싱해야 한다. `05_도메인_사전.md`에 해독 규칙이 있으니 그걸 따른다.

또 이 자료는 소설 본문을 포함하지 않는다. 소설 집필/수정 시 원본 HTML을 직접 편집해야 한다.

원본 대화 4호~6.1호도 포함하지 않는다. 세부가 필요하면 해당 파일을 `grep`으로 직접 검색.

## 마지막 한 가지

천홍이 이 자료를 보고 "이거 빠졌어" 또는 "이건 틀렸어"라고 지적할 가능성이 높다. 그건 **기대되는 피드백이다**. 이식 자료는 완벽할 수 없고, 피드백을 받으며 갱신되는 문서다.

천홍의 지적을 받으면:
- 해당 문서를 즉시 수정.
- 수정 사유와 변경 내용을 기록.
- 동일한 정보가 여러 문서에 있으면 모두 동기화.
